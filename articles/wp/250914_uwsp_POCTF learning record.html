Reverse 300-2 Make the Pieces Sing

Desc

ANOTHER musical challenge?

This time I‚Äôm going to provide you an application and a validator script. Your job is to use the both of them to reconstruct the very special password I used. That password? A MIDI file. Yes. The password is a song.

SING
MD5: 1A82AB11D396DEF82EC09AE063CD280F
Validator

The validator will print pairs, pitch, and a hex string. This hex string is¬†550[...]E60. If you get stuck, you can send the string to Prof. Johnson to at least confirm that step is correct.

HINT:
What SING actually hashes and uses as the key isn‚Äôt ‚Äúthe whole MIDI,‚Äù it‚Äôs a 40-byte fingerprint built only from the first 20 Note-On events it sees (non-drum channels only). After merging tracks and sorting by time, it keeps at most one note per tick (the highest pitch at that tick). From those 20 onsets it builds the byte sequence.
That 40-byte sequence is HKDF‚Äôed (SHA-256, salt ‚Äúmusic-key-v1‚Äù, info ‚ÄúREV300-2‚Äù) into the AES-256-GCM key. The validator‚Äôs ‚Äúpairs, pitch, and a hex string‚Äù is literally this 40-byte fingerprint in hex. The fact it begins with 55‚Ä¶E60 means you already know the start/end of the exact bytes SING will consume.
So the game isn‚Äôt to compose a real song‚Äîit‚Äôs to craft a minimal MIDI whose first 20 Note-On events (on channels ‚â† 9) produce that exact 40-byte sequence. You have complete freedom to make the MIDI ‚Äúugly‚Äù as music; SING only cares about those bytes.

What else‚Ä¶
Use channel 0 (status 0x90) with any nonzero velocity (e.g., 0x40). Channel 9 (10th) is drums and ignored.
Make sure each of your 20 Note-On events happens at a distinct tick. If you accidentally place two Note-Ons at the same tick, SING keeps only the highest pitch at that tick.
The actual tick values are only relevant mod 256. To realize a desired dt byte, just set the delta-time to that value (or that value plus any multiple of 256).
Include quick Note-Offs (or Note-On with velocity 0) in between so most DAWs won‚Äôt complain, but SING ignores them anyway.
Format 0 or 1 is fine. A single track (format 0) is simplest. Any reasonable PPQ (e.g., 480) works.

HINT 2:
It‚Äôs also possible to just decrypt (or derive the song) with the full key. Here it is: 550027202760306057602cc02cc050c027c02c60576029202960306059602ee05c605a6059602e60

Your goal: escalate the privileges of the running service process and read the flag stored at /root/flag, then return it to your client.
